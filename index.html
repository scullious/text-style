Стандарты кодинга на PHP

1. Общие положения

  • Разрешены только php-теги <?php и <?=

  • В качестве отступов ДОЛЖНЫ быть использованы 4 пробела, никаких табуляций

  • НЕТ СТРОГИХ ограничений по длине строки, но она ДОЛЖНА быть до 120 символов, в идеале – 80 символов

  • Пустые линии МОГУТ быть добавлены для улучшения читаемости кода

  • В одной линии ДОЛЖНО содержаться не более одного выражения

  • Все ключевые слова и служебные константы true, false и null ДОЛЖНЫ быть нижнего регистра

  • Комментарии начинаются с двойного слеша // , после которого следует 1 пробел

  • Рекомендуется называть все компоненты кода в соответствии с их предназначением. Этот подход улучшается читаемость кода

2. Потоки управления, условия

  • ДОЛЖЕН быть 1 пробел до (

  • НЕ ДОЛЖНО быть пробелов после (

  • НЕ ДОЛЖНО быть пробелов до )

  • ДОЛЖЕН быть 1 пробел после )

  • Вместо else if следует использовать elseif

  • КРАЙНЕ желательно использовать операторы строгого тождества/несоответствия (===, !==) вместо обычного равенства/неравенства (==, !=)

  • По обеим сторонам от операторов в условиях ДОЛЖНО быть по 1 пробелу

  • Логические операторы И, ИЛИ записываются в виде && , ||

  • Перед открыв. фигурн. скобкой { НЕ ДОЛЖНО быть отступов, только 1 пробел

  • После определения условия, начала цикла и т.п. после { с новой строки начинается тело оператора с отступом

  • По окончании написания тела, с новой строки и без отступов относительно строки объявления оператора ставится }

    • if ($a === $b) {
    // ...
} elseif ($a !== 0) {
    // ...
} else {
    // ...
}

  • Тернарные операторы нельзя вкладывать друг в друга. С обеих сторон от операторов ? и : ДОЛЖНО быть по 1 пробелу

    • $var = 3;
$isTrue = ($var > 0 ? false : true); // returns false
    $message = 'Hello '.($user->isLoggedIn() ? $user->getName('first_name') : 'Guest');

  • В конструкциях switch-case условия case ДОЛЖНЫ быть с отступом от switch, тело case ДОЛЖНО быть с отступом от case, а break ДОЛЖНО быть на одном уровне с телом case. Если тело case не пустое и необходимо отрабатывание кода, в конце ДОЛЖЕН быть комментарий // no break

    • switch ($expr) {
    case 0:
        echo 'First case, with a break':
        break;
    case 1:
        echo 'Second case, which falls through';
        // no break
    case 2:
    case 3:
    case 4:
        echo 'Third case, return instead of break';
        return;
    default:
        echo 'Default case';
}

  • Конструкция for выглядит следующим образом

    • for ($i = 0; $i < 10; $i++) {
    // for body
}

  • Конструкция foreach выглядит следующим образом

    • foreach ($iterable as $key => $value) {
    // foreach body
}

  • Блок try-catch выглядит следующим образом

    • try {
    // try body
} catch (FirstExceptionType $e) {
    // catch body
} catch (OtherExceptionType $e) {
    // catch body
}

3. Функции и замыкания

  • Имя функции ДОЛЖНО быть в форматe camelCase

  • При объявлении и вызове функции между ее именем и открывающейся круглой скобкой НЕ ДОЛЖНО быть пробелов

  • В круглых скобках перед параметрами/аргументами НЕ ДОЛЖНЫ быть пробелов, после них идет запятая, за которой ДОЛЖЕН стоять 1 пробел

  • Аргументы с дефолтными значениями ДОЛЖНЫ быть в конце списка аргументов

  • Перед открыв. фигурн. скобкой { НЕ ДОЛЖНО быть отступов, только 1 пробел

  • После открыв. фигурн. скобки начинается тело функции с новой строки с отступом

    • function foo($arg1, $arg2) {
    // body
}

  • Список аргументов МОЖЕТ быть разбит в несколько строк. Каждый пункт ДОЛЖЕН начинаться с новой строки с отступом

  • Когда список заканчивается, закрывающаяся круглая скобка ДОЛЖНА быть с новой строки на уровне объявления замыкания. Далее между круглой и фигурной скобкой ДОЛЖЕН быть 1 пробел

    • function bar(
    $arg1,
    $arg2,
    $arg3,
    $arg4
) {
    // body
}

  • Замыкания ДОЛЖНЫ быть объявлены после слова function через пробел, а также ДОЛЖНЫ быть пробелы до и после слова use

  • Открывающаяся фигурная скобка ДОЛЖНА быть в строке объявления замыкания, а закрывающаяся скобка - на следующей строке после тела замыкания, после скобки ДОЛЖНА идти точка с запятой ;

  • Замыкания объявляются следующим образом

    • $closureWithArgs = function ($arg1, $arg2) {
    // body
};
    $closureWithArgs = function ($arg1, $arg2) use ($var1, $var2) {
    // body
};

  • Список аргументов/переменных МОЖЕТ быть разбит в несколько строк по аналогии с функциями

    • $longArgs_noVars = function (
    $longArgument,
    $longerArgument,
    $muchLongerArgument
) {
    // body
};
    $longArgs_noVars = function () use (
    $longVar1,
    $longerVar2,
    $muchLongerVar3
) {
    // body
};
    $longArgs_noVars = function (
    $longArgument,
    $longerArgument,
    $muchLongerArgument
) use (
    $longVar1,
    $longerVar2,
    $muchLongerVar3
) {
    // body
};

4. Классы, методы и свойства

  • К термину “класс” относятся все классы, интерфейсы и т.п.

  • Все служебные слова ДОЛЖНЫ быть нижнего регистра

  • Имена классов ДОЛЖНЫ быть именами существительными и начинаться с большой буквы. Если имя состоит из нескольких слов, то все слова также с большой буквы: StudlyCase

  • Имена интерфейсов ДОЛЖНЫ быть именами прилагательными

  • Имена методов ДОЛЖНЫ быть либо глаголами, либо сочетанием глагол-существительное и начинаться с маленькой буквы, каждое последующее слово – с большой: camelCase

  • Имена свойств классов и переменных ДОЛЖНЫ быть именами существительными и начинаться с маленькой буквы, каждое последующее слово – с большой

  • Имена констант ДОЛЖНЫ быть именами существительными. Все буквы в имени – верхнего регистра, а между словами должны стоять знаки нижнего подчеркивания. (Обычно константы объявляются с public static или final модификаторами)

    • const DATE_APPROVED = '2012-06-01';

  • После объявления классов и методов открыв. фигурная скобка ДОЛЖНА переходить на следующую строку без отступов

  • Модификаторы классов пишутся в одну строку без переносов

    • class ClassName extends ParentClassName implements Interface
{
    // Class definition
}

  • Если интерфейсов слишком много, то каждый ДОЛЖЕН быть перенесен на следующую строку с отступом

    • class ClassName extends ParentClassName implements
    Interface1,
    Interface2,
    Interface3,
    Interface4
{
    // Class definition
}

  • Все методы без исключения ДОЛЖНЫ содержать модификаторы

  • Методы и свойства ДОЛЖНЫ быть в следующем порядке: сначала свойства final, public, protected, private, static, затем методы abstract/final, public, protected, private, static

  • При вызове метода или функции между именем и скобками НЕ ДОЛЖНО быть пробелов. С двух сторон от -> и :: НЕ ДОЛЖНО быть пробелов

    • bar();
$foo->bar($arg1);
Foo::bar($arg2, $arg3);
